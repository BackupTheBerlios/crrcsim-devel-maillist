Subject: [Crrcsim-devel] New release: CRRCsim 0.9.11
Message-ID: <20100602235011.d3d9b243.jan.reucker@web.de>

The CRRCsim dev team released version 0.9.11 of the cross-platform R/C model aircraft simulator today. Besides lots of bug fixes, several interesting new features were implemented: Support for simulating multicopters, driver for CT6A transmitters, adjustable model volume, crash detection, a flight recorder that also works in the F3F mode and a stable scenery file format.

Download link for source package and Win32 installer:

http://crrcsim.berlios.de/wiki/index.php?n=CRRCsim.DownLoad

Have fun!




Release Notes for CRRCSim 0.9.11
--------------------------------
- Added command line option -V (print version info, configuration options and exit).
- removed obsolete files from distribution
- hitting any unused key causes CRRCSim to try reloading the FDMs parameters
- Fixed bug #15694 Segfault when configuring audio interface (patch provided by Glenn Trigg)
- Improved helicopter parameters
- New FDM including an example model: multicopter
- New CT6A transmitter interface (by Nikolay B. Eremeyev)
- Implemented real engine downthrust, adjusted Flexifly accordingly
- Repaired flap and spoiler parameters
- Implemented feature request #4271: Model volume can now be configured through the GUI
- Heli and Multicopter: reversed nick input to be like fixed wing
- Fixed bug #16552 Collision detection with multiple objects fail
- Fixed bug #16204 core dump on karmic. Fixed CMake build with libCGAL.
- Simulate influence of retractable gears on aerodynamics
- Implemented rudimentary crash handling with a new spring.max_force attribute
- F3F on all scenery
- Implemented scenery file specification v3 
- Feature Request #4311 (Replay tool - a shadow plane): You can now save and replay
  your flights and race against a shadow plane in F3F mode.
- Fixed bug #017127 Selections are hardly visible in the GUI
- Fixed bug #017076 Build error when CGAL installed on Ubuntu karmic
- Implemented feature request #4470 Improve scenery selection dialog

Date: 2010-Jun-09 23:24
By: jreucker

Comment:
Karsten,

indeed, interesting results! And from your logs I can see why it crashed and
what was "cured": The original 0.9.11 tried to open the scenery

/usr/share/crrcsim/scenery/davis-orig.xml

which seems to be on your computer although this directory is not part of the
search path. Did you have an older Ubuntu package on your computer before?
This would explain this, because Ubuntu packages use /usr instead of /usr/local
to install CRRCsim.

However, the location dialog in 0.9.11 does not scan this directory any more,
so it does not find this scenery. This lead to the crash, because CRRCsim tried
to set the "current" (highlighted) entry of the location listbox to a non-existing
index. The tip version performs an extra check before doing this, so it won't
crash anymore. You get the scenery dialog, select a new scenery, this one is
stored in your config file on exit. Now you start up 0.9.11 again, it finds
the scenery from your config file and everything is allright.

I guess I'll add some extra sanity checking to make this kind of bullet-proof...

Many thanks for your support!

(Just a side note: I guess this could have been "fixed" by deleting the configuration file ~/.crrcsim/crrcsim.xml and reconfiguring CRRCsim, but I'm glad that you helped me find the real reason for the crash!)

By: jreucker

Comment:
Karsten,

indeed, interesting results! And from your logs I can see why it crashed and
what was "cured": The original 0.9.11 tried to open the scenery

/usr/share/crrcsim/scenery/davis-orig.xml

which seems to be on your computer although this directory is not part of the
search path. Did you have an older Ubuntu package on your computer before?
This would explain this, because Ubuntu packages use /usr instead of /usr/local
to install CRRCsim.

However, the location dialog in 0.9.11 does not scan this directory any more,
so it does not find this scenery. This lead to the crash, because CRRCsim tried
to set the "current" (highlighted) entry of the location listbox to a non-existing
index. The tip version performs an extra check before doing this, so it won't
crash anymore. You get the scenery dialog, select a new scenery, this one is
stored in your config file on exit. Now you start up 0.9.11 again, it finds
the scenery from your config file and everything is allright.

I guess I'll add some extra sanity checking to make this kind of bullet-proof...

Many thanks for your support!

(Just a side note: I guess this could have been "fixed" by deleting the configuration file ~/.crrcsim/crrcsim.xml and reconfiguring CRRCsim, but I'm glad that you helped me find the real reason for the crash!)
