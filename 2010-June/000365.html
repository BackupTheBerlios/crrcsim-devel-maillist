Summary: Wrong starting position when loading scenery from cmdline

Details: When loading a scenery file from the command line (-l option), the starting position for the model is not set correctly.

Example to reproduce the bug:

./crrcsim -l scenery/davis-orig.xml models/gap65.xml
<exit and save>
./crrcsim -l scenery/cape_cod-orig.xml

The plane will now start far below ground (and therefore crash immediately).

Side note: loading a new plane at this point will not reset the simulation. AFAIK this has always been different in previous versions? Now it takes a manual simulation reset to launch the plane.

CRRCsim 0.9.11, Linux
